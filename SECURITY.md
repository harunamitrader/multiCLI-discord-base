# セキュリティ

## 最初に読んでください

このプロジェクトは、セキュリティを厳密に保証する製品ではありません。

これはローカル bridge ツールであり、次のことができます。

- プロンプトや添付ファイルを Codex CLI に渡す
- 会話内容を Discord に送る
- 監視フォルダ内の変更ファイルを Discord に送る
- 設定次第で強いローカル権限で Codex を動かす

設定や運用を誤ると、次のような情報が漏れる可能性があります。

- ソースコード
- ドキュメント
- スクリーンショット
- 監視対象フォルダ内の機密ファイル
- Discord Bot Token
- 会話履歴

## 特に注意が必要な設定

以下は特に影響が大きい設定です。

- `CODEX_ENABLE_SEARCH=true`
- `CODEX_BYPASS_APPROVALS_AND_SANDBOX=true`
- `FILE_WATCH_ENABLED=true`
- 範囲が広すぎる `CODEX_WORKDIR`
- 範囲が広すぎる `FILE_WATCH_ROOT`

これらによって起こり得ること:

- 想定外の情報流出
- 強すぎる権限でのコマンド実行
- Discord への不要なファイル投稿

## 安全な使い方の推奨

このプロジェクトは、次の条件を満たす場合にのみ使うことを推奨します。

- 自分が管理する Windows PC で使う
- 自分が管理する Discord サーバーで使う
- 書き込みできる人を信頼できるチャンネルだけで使う
- 保存先や送信先を理解したうえで使う
- 秘密情報が漏れた場合に再発行や停止対応ができる

推奨事項:

- Discord サーバーは私用または厳密管理されたものだけを使う
- Bot のアクセス先は最小限のチャンネルに絞る
- 可能なら `DISCORD_ALLOWED_CHANNEL_IDS` を設定する
- `.env` は Git に入れない
- パスワード、鍵、ブラウザプロファイル、個人情報を含むフォルダを監視対象にしない
- 可能なら専用 Windows アカウントや専用 PC を使う

## このプロジェクトが守ってくれないこと

現時点で、次の仕組みはありません。

- 強い認証
- 厳密な権限管理
- 自動秘密情報マスキング
- 侵害された Discord アカウントへの対抗
- 侵害されたローカル Windows アカウントへの対抗

## Discord 連携のリスク

Discord 連携には追加のリスクがあります。

- 紐付いたチャンネルで発言できる人はセッションに影響を与えられます
- 添付ファイルが Discord 履歴に残る場合があります
- Bot Token が漏れると Bot を操作される可能性があります
- サーバー設定を誤ると内部情報が見える相手が増えます

公開サーバーや不特定多数が出入りするサーバーでは使わないでください。

## ファイル監視のリスク

ファイル監視を有効にすると:

- 変更されたファイルが Discord に投稿されることがあります
- 監視フォルダの選び方が非常に重要になります

次のような場所は監視対象にしないでください。

- ユーザープロファイル全体
- ブラウザの保存データ
- パスワード管理ツールの保存先
- 個人ファイルと仕事ファイルが混ざった同期フォルダ
- 許可のない業務用文書フォルダ

## ローカル保存のリスク

次の場所には重要情報が入る可能性があります。

- `.env`
- `data/bridge.sqlite`
- `data/uploads/`
- ログファイル

これらは秘密情報として扱ってください。

## ネットワーク公開について

この bridge はローカル利用を前提にしています。

- `HOST` は基本的に `127.0.0.1` のまま使ってください
- LAN やインターネットへ公開しないでください
- リバースプロキシの公開先にしないでください

## オープンソース公開前の確認

公開前に最低限これを確認してください。

- `.env` をコミットしていない
- `data/` をコミットしていない
- 添付ファイルやログをコミットしていない
- Discord Bot Token が履歴に残っていない

## 脆弱性報告について

現時点では専用の非公開連絡先を定義していません。

そのため、GitHub Issue などで報告する場合は、トークン・個人情報・機密ファイル・DB 内容を絶対にそのまま貼らないでください。必要なら先に伏せ字化してください。
